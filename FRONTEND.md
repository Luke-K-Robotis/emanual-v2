ROBOTIS e-Manual (Docusaurus 기반) 프론트엔드 UI 구성과 자산/빌드 사이즈 최적화의 운영 가이드. 다른 부서/담당자가 작업을 이어받을 때 참고할 자료.

- 사이트: https://docs.robotis.com
- Repo: https://github.com/Luke-K-Robotis/emanual-v2 (master)
- Docusaurus: 3.10.x (classic preset)
- Node: 20+
- Locale: en (기본), ko (ja 는 v1 직전 삭제)
- 1차 완료 태그: `v1.0-phase1` (이관 시점 스냅샷)

#### Doc id 규칙 (필수 인지)

페이지의 frontmatter `id`가 명시되면 doc id가 `<dir>/<id>` 형태가 된다. 예: `dxl/x/index.mdx` 의 `id: x` → doc id = `dxl/x/x`. 사이드바에서 `link.type: 'doc'` 으로 매칭할 때 이 점에 주의.

> 단, **URL** 은 frontmatter id 와 무관하게 `index.mdx` 의 경우 폴더 URL(`/docs/dxl/x`)로 매핑된다. mega-menu href 는 doc id 가 아닌 **실제 URL** 을 써야 한다 (예: `/docs/dxl/x/x` 는 404, `/docs/dxl/x` 가 정답).

### 2.6 Mega-menu (DYNAMIXEL, Systems)

순수 React 컴포넌트가 아닌 **`type: 'html'` 인라인 HTML** + **vanilla JS sticky** 패턴.

- HTML 구조는 `docusaurus.config.ts` 의 `items: [{type: 'html', value: '<div class="mega-menu">…</div>'}]` 안에.
- DYNAMIXEL: 좌측 카테고리 3개 (Quick Start / DYNAMIXEL / Protocol).
- Systems: 좌측 카테고리 4개 (Humanoid / Robot Hand / Manipulator / **TurtleBot3**). TurtleBot3 패널은 단일 항목이며 `/docs/category/turtlebot3` (generated-index) 으로 연결.
- 활성화 동작: `src/clients/mega-menu-sticky.js`
  - 카테고리 `mouseenter` / `focus` 시 `--active` 클래스 토글.
  - `mouseleave` 핸들러 없음 → 한 번 활성화된 panel 은 다른 카테고리로 가기 전까지 sticky. 마우스가 우측 영역으로 이동해도 panel 표시 유지.
  - `MutationObserver` 로 SPA 라우팅 후 새 mega-menu DOM 도 자동 init.
- CSS는 `custom.css` 의 `.mega-menu*` 클래스. `:has()` 도 정의되어 있지만 fallback 용. 실제 활성/비활성은 JS가 부여하는 `--active` 클래스로 제어.

> **mega-menu href 작성 규칙**: 반드시 브라우저가 실제로 가는 URL을 사용. `index.mdx`(또는 frontmatter id ≠ 폴더명) 인 페이지를 doc-id 형식 (`/docs/dxl/x/x`) 으로 잘못 적으면 404. 폴더 URL(`/docs/dxl/x`)이 정답. PR 검토 시 mega-menu URL 변경분은 로컬 빌드 후 `build/docs/...` 에 실제 파일 생성 여부로 검증할 것.

## 3. 자산 최적화 워크플로 (스크립트 카탈로그)

`scripts/` 안의 운영 도구. 모두 Node 20+ + Docusaurus 디렉터리 기준. 카테고리별:

- **변환 (1회성, 보존용)**: `convert-*.js` (16개) — 마이그레이션 당시 Jekyll md → Docusaurus mdx 일괄 변환에 쓰임. 새 콘텐츠에는 사용하지 않지만 향후 추가 마이그레이션 발생 시 패턴 참고용으로 보존.
- **자산 압축 (반복 운영)**: `png-to-webp.js`, `jpg-to-webp.js`, `rewrite-png-references.js`, `rewrite-jpg-references.js`, `measure-png-compression.js`
- **빌드 후처리**: `dedupe-i18n-static.sh` (CI에서 자동 실행)
- **redirect**: `build-redirects.js`
- **헤딩 anchor**: `inject-heading-anchors.js`, `inject-anchors-rna.js`, `fix-heading-anchors.js`, `hide-popup-sidebar.js`
- **링크/자산 복원**: `restore-internal-links.js`, `recover-missing-images.js`

### 3.1 이미지 압축

이미지 추가/갱신 후 두 단계로 진행.

```bash
# 1) PNG → WebP q=85 (alpha 보존)
node scripts/png-to-webp.js

# 2) JPG → WebP q=85
node scripts/jpg-to-webp.js

# 3) mdx 및 소스 파일의 image src 확장자 일괄 치환
node scripts/rewrite-png-references.js
node scripts/rewrite-jpg-references.js
```

> 두 rewrite 스크립트는 **실제 .webp 파일이 `static/img/` 에 존재할 때만** 경로를 교체한다. 따라서 변환 직후 실행하면 안전.

**압축률 사전 측정**: `node scripts/measure-png-compression.js` — 200개 샘플로 JPEG q=80/85, WebP lossless/q=80/q=85 변환 비율을 추정.

| 형식 | PNG 대비 비율 (q=85, 측정값) | 비고 |
|---|---|---|
| JPEG q=85 | 14.2% | alpha 손실 |
| WebP lossless | 44.7% | alpha 보존 무손실 |
| WebP q=85 ✅ | 10.3% | alpha 보존 + 안전 화질, **권장** |

실측 효과: PNG 630 MB → 147 MB (77% 절약), JPG 192 MB → 62 MB (68% 절약).

### 3.2 i18n 빌드 중복 제거 (`scripts/dedupe-i18n-static.sh`)

Docusaurus i18n 빌드는 `build/<locale>/img/` 에 root `build/img/` 와 동일한 자산을 중복 복사한다. CI 후처리로 제거.

- `build/ko/img/` 삭제
- ko HTML 의 `/ko/img/` → `/img/` 일괄 sed
- `build/ko/search-index.json` 도 동일 치환

`.github/workflows/deploy.yml` 의 build job 마지막 step에 등록되어 있으니 수동 실행은 보통 불필요.

### 3.3 redirect 자동 생성 (`scripts/build-redirects.js`)

원본 Jekyll 의 모든 `permalink:` 값을 스캔해 Docusaurus client-redirects 가 사용할 `redirects.generated.json` 생성.

- target 페이지가 `docusaurus/docs/` 에 실제 존재할 때만 emit (점진적 활성화).
- en locale 만 emit (ko/ja path 는 plugin 검증 한계로 제외).
- `PERMALINK_OVERRIDES` 로 RH-P12-RN 같이 특수 매핑 처리.

콘텐츠가 추가될 때마다 다시 실행해야 새 redirect 가 생성된다.

### 3.4 헤딩 anchor 처리

- `scripts/inject-heading-anchors.js <series>` — 원본 `<a name="X">` 정보를 변환된 mdx 헤딩 직전에 `<a id="X"></a>` 로 부착 (idempotent).
- `scripts/fix-heading-anchors.js` — 헤딩 line 안의 markdown self-link `[Foo](#foo)` / inline `<a name>` 제거. nested `<a>` SSG warning 방지.

### 3.5 내부 링크 복원

`scripts/restore-internal-links.js` — 변환 중 미완 영역 보호용으로 `https://emanual.robotis.com/docs/...` 외부 URL 로 강등됐던 링크들을 `/docs/...` 내부 링크로 복원 (target 페이지가 docs 트리에 실제 존재할 때만).

### 3.6 누락 자산 복원

`scripts/recover-missing-images.js` — 모든 mdx 의 `/img/*` 참조를 스캔, 누락된 파일은 `source/assets/images/` 에서 (대소문자 무관) 찾아 복사. backslash path 정규화도 같이.

## 4. 빌드 사이즈 한계 관리

GitHub Pages artifact 한계는 **1 GB**. 운영 중 빌드 결과 사이즈를 항상 1 GB 안에 유지해야 한다.

### 4.1 현재 마진

```
빌드 단계별 사이즈 (raw)
3.7 GB (원본 PNG/JPG, 3 locale)
↓ dedupe-i18n-static
2.3 GB
↓ PNG → WebP q=85
1.54 GB
↓ JPG → WebP q=85
1.345 GB
↓ ja locale 제거 (현재 상태)
0.949 GB ✅
```

GitHub Actions 의 artifact 업로드는 gzip 압축되므로 **실제 업로드 사이즈는 약 500–600 MB** 수준이 된다.

### 4.2 새 자산 추가 시 체크리스트

1. 가능하면 **WebP** 로 저장 (alpha 있으면 q=85, 없으면 q=80).
2. 큰 이미지는 200–800 KB 이내로 압축.
3. `npm run build` 결과 `du -sh docusaurus/build` 1 GB 이내인지 확인.
4. 1 GB 근처면 호스팅 변경(Cloudflare Pages 25 GB) 검토.

### 4.3 1 GB 초과 시 대안

| 방법 | 효과 | 비고 |
|---|---|---|
| 추가 이미지 압축 | 작음 (이미 WebP) | `scripts/jpg-to-webp.js`, sharp 옵션 q=75 |
| 새 locale 추가 보류 | locale 1개당 약 400 MB | webpack chunks가 locale별 별도 |
| 외부 CDN (이미지 분리) | 큼 (500 MB+) | `static/img/` 를 Cloudflare R2 등으로 |
| Cloudflare Pages 이전 | 한계 25 GB | DNS만 변경, repo/도메인 유지 |

## 5. 운영 자주 묻는 작업

### 5.1 새 페이지 추가

1. `docusaurus/docs/<area>/<name>.mdx` 작성 (frontmatter `id`, `title`, `sidebar_label`).
2. 같은 path 의 ko 미러: `docusaurus/i18n/ko/docusaurus-plugin-content-docs/current/<area>/<name>.mdx` (없으면 영문 fallback).
3. `sidebars.ts` 의 해당 sidebar 트리에 등록 (autogenerated 면 자동).
4. `node scripts/build-redirects.js` 로 redirect 갱신.
5. `npm run build` 로 검증.

### 5.2 새 navbar mega-menu 항목 추가

1. 썸네일을 `static/img/mega-menu/<name>.webp` (200×200 정사각, 투명).
2. `docusaurus.config.ts` 의 해당 mega-menu HTML 에 `<a class="mega-menu__product"…><img src="/img/mega-menu/<name>.webp" …/><span>이름</span></a>` 추가.
3. **href는 실제 페이지 URL로**. `index.mdx` 인 페이지면 `/docs/<폴더>/` (frontmatter id 가 폴더명과 달라도 URL 은 폴더 그대로). 카테고리 인덱스(generated-index)면 `/docs/category/<카테고리-slug>` (slug 는 라벨에서 자동 lowercase + space→hyphen).
4. 새 카테고리 인덱스 페이지가 필요하면 `sidebars.ts` 에 `{type:'category', label:'…', link:{type:'generated-index'}, items:[…]}` 형태로 사이드바 루트를 감싸야 한다 (예: `turtlebot3Sidebar`).
5. 새 제품 폴더면 `sidebars.ts` 에 `{type:'autogenerated', dirName:'platform/<name>'}` 사이드바 1줄 추가 — 그래야 진입 시 platformSidebar(공통) 대신 제품 전용 사이드바가 뜬다.
6. 로컬 빌드 후 `build/docs/<해당-경로>/index.html` 생성 여부로 404 검증.

### 5.3 사이드바 카테고리 라벨 한국어화

`i18n/ko/docusaurus-plugin-content-docs/current.json` 의 `sidebar.<sidebarId>.category.<Label>` 키. 자동 추출은:

```bash
cd docusaurus
npx docusaurus write-translations --locale ko
```

### 5.4 i18n 텍스트 추가

소스에서 `<Translate id="…">텍스트</Translate>` 사용 후 `write-translations` 실행. ko 번역은 `i18n/ko/code.json` 에서 수동 입력 (자동 번역 금지 정책).

### 5.5 색상/디자인 변경

대부분 `src/css/custom.css` 의 CSS 변수만 수정하면 사이트 전반에 반영된다. 그라데이션이 직접 들어간 곳 두 군데(`index.module.css` 의 hero 와 CTA shadow)는 같이 손볼 것.

### 5.6 favicon / 로고 교체

- favicon: `docusaurus/static/img/favicon.ico` 만 교체.
- 로고: `docusaurus/static/img/emanaul_logo.webp` 교체. width 는 `custom.css` 의 `.navbar__logo img { width: 150px }` 로 고정.

## 6. CI / 배포 (`.github/workflows/deploy.yml`)

- 트리거: master push (paths `docusaurus/**` 또는 `.github/workflows/deploy.yml`), workflow_dispatch
- 환경: Ubuntu, Node 20, npm ci
- 빌드 환경변수: `DOCUSAURUS_IGNORE_SSG_WARNINGS=true` (cosmetic SSG 워닝의 빌드 차단 방지)
- 빌드 후처리: `bash scripts/dedupe-i18n-static.sh` (1.4 GB 절약)
- 업로드: `actions/upload-pages-artifact@v3` (path: `docusaurus/build`)
- 배포: `actions/deploy-pages@v4`

`docusaurus.config.ts` 의 빌드 옵션:
- `onBrokenLinks: 'throw'` (필수 — 깨지면 빌드 실패)
- `onBrokenAnchors: 'ignore'` (`<a id>` invisible anchor가 Docusaurus 검증 알고리즘에 안 잡혀서 끔. 실제 anchor scroll 은 정상)
- `onBrokenMarkdownLinks: 'warn'` / `onBrokenMarkdownImages: 'warn'`

## 7. 알려진 한계 / 후속 작업

1. **다국어 redirect**: 현재 en URL만 redirect. kr/jp 원본 URL → ko 페이지 redirect 는 client-redirects 플러그인의 valid-path 검증 한계 (카테고리 인덱스 인식 못 함) 로 보류. 호스팅 레벨 redirect 또는 별도 미들웨어로 후속.
2. **ko 본문 영문 잔재**: 대량 마이그레이션 페이지 일부에 영문 fallback 이 남아있다 (kr 원본 부재 페이지). 자동번역 금지 정책상 ROBOTIS 도메인 표기로 수동 채워야.
3. **broken anchor 워닝 (silent)**: `onBrokenAnchors: 'ignore'` 로 로그 정리. 시각 검증으로 실제 broken 발견 시 case-by-case.
4. **GitHub Pages 1 GB 한계**: 현재 949 MB 로 통과 가능. 향후 콘텐츠 증가시 호스팅 변경 (Cloudflare Pages 권장) 검토.
5. **DYD 시리즈 누락**: 원본에 DYNAMIXEL DRIVE 시리즈가 있지만 `source/docs/en/dxl/` 에 별도 폴더 없어 변환 누락.
6. **mega-menu thumbnail 일부 표기**: `robotos-op.webp` (의도 표기), `rh-p12-rn-ur.webp` 미준비 (현재 `rh-p12-rn.webp` 공유), `Y-series.webp` 대문자. turtlebot3 추가 완료.
7. **R+ 1.0 / 2.0 / 3.0 / Mobile App 콘텐츠 삭제됨**: Software 카테고리에서 4종 폴더(`docs/software/rplus1`, `rplus2`, `rplustask3`, `rplus_mobile`, `mobile_app`)를 제거. 본문/사이드바에서 참조하던 모든 로컬 링크 `/docs/software/rplus*`, `/docs/software/mobile_app/*` 는 외부 URL `https://emanual.robotis.com/docs/<en|kr>/software/...` 로 강등되어 있다. 향후 이 도메인이 폐쇄되면 dead link 가 되므로 `grep -r emanual.robotis.com/docs docusaurus/docs docusaurus/i18n` 으로 일괄 점검 필요.
